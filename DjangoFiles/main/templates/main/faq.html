Commands

Command Argument Description
info This will return detailed information about the host machines network, physical drives, processor, graphics chip and more.
check This will return the current permissions.
where This will return the current working directory.
cd <target_path> This will return the change the current working directory to the given target path.
ls This will list out all directories and files in the current working directory. It will also display hidden files.
cat <target_file> This will read out the content of the target file via standard output.
rm <target_file> This will delete the target file.
cp <source_file> <destination_path+destination_filename> This will copy the target file to the desired location. You must include the file name of the destination file.
mkdir <directory_name> This will create a new directory within the current working directory.
removedir <target_directory> This will recursively delete the target directory.
ps This will list out all active processes.
get <target_file> This will download the target file in the given current working directory.
arp This will return ARP tables.
nestat This will return all listening/open ports. No duplication.
allWindows This will return the processes that currently have an active open window.

What is a C2?

A C2 (Command and Control) framework is a software tool or platform designed to enable remote control and management of compromised systems or networks. It provides a means for attackers to communicate with and control compromised machines, typically for malicious purposes.

C2 frameworks are often used by cybercriminals, hackers, or advanced persistent threat (APT) groups to maintain persistence, conduct reconnaissance, execute commands, and exfiltrate data from compromised systems. These frameworks typically consist of two main components: a client and a server.

The client component, also known as an agent or implant, is typically installed on the compromised system or device. It acts as a backdoor, allowing the attacker to maintain control over the system and execute various commands remotely. The client is designed to be lightweight, stealthy, and capable of evading detection by security tools.

The server component, also referred to as the C2 server or command center, is where the attacker manages the compromised systems. It receives communications and commands from the client agents, and it provides a control interface for the attacker to interact with the compromised systems. The server component is often hosted on a remote server or infrastructure controlled by the attacker.

The communication between the client and server components of a C2 framework is typically conducted over various channels, such as HTTP(S), DNS, or other covert channels, to avoid detection by network security solutions. Encryption and obfuscation techniques may also be employed to make the communication traffic harder to analyze.

C2 frameworks are designed to provide flexibility and scalability to attackers, allowing them to control multiple compromised systems simultaneously, issue commands, gather information, and coordinate attacks. These frameworks often have features such as file transfer capabilities, keylogging, screen capture, and the ability to execute arbitrary code on compromised systems.

It's important to note that C2 frameworks are primarily used by malicious actors and are illegal to use without proper authorization. Cybersecurity professionals and researchers may also use similar frameworks for defensive purposes, such as analyzing and understanding the techniques used by attackers to develop effective countermeasures.
